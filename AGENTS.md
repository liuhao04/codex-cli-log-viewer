# AiCliLog Project Instructions

AiCliLogApp is a standalone macOS project for `AiCliLog.app`, a local log
viewer for Codex CLI and Claude Code sessions.

## Project Context

- Treat this directory as its own project root:
  `/Users/liuhao/Library/Mobile Documents/com~apple~CloudDocs/Projects/ai-cli-log`
- Do not rely on the former nested path under `set-codex`.
- Preserve dirty worktrees. Never revert user changes unless explicitly asked.
- Keep Claude-specific files if any are added later; Codex and Claude workflows
  should be able to coexist.

## App Identity

- Product name: `AiCliLog`
- Installed app path: `/Applications/AiCliLog.app`
- Bundle identifier: `io.github.aiclilog`
- Main source file: `Sources/AiCliLogApp/AiCliLogApp.swift`
- App metadata: `App/Info.plist`
- Xcode project: `AiCliLogApp.xcodeproj`

## Build And Install

Use the repo script for normal verification and installation:

```bash
Scripts/install-app.sh
```

The script builds the `AiCliLog` scheme, replaces the app in
`/Applications`, refreshes Launch Services, and opens the app.

For build-only checks:

```bash
xcodebuild -project AiCliLogApp.xcodeproj -scheme "AiCliLog" -configuration Debug -destination 'platform=macOS' build
```

## Versioning Rule

When changing app code, app resources, Xcode project settings, install behavior,
or user-visible documentation for the app:

- Bump the app version/build as appropriate in both `App/Info.plist` and
  `AiCliLogApp.xcodeproj/project.pbxproj`.
- After a successful build, report the resulting marketing version and build
  number.
- Automatically run `Scripts/install-app.sh` and restart/open the app before the
  final response.
- If install or restart fails, report the failure clearly and include the exact
  command that failed.

Pure project housekeeping that does not affect the app bundle, such as moving
the repository or updating this `AGENTS.md`, does not require a version bump or
app reinstall.

## Data And Privacy

- The app reads local Codex data from `~/.codex/state_5.sqlite` and
  `~/.codex/sessions/**/rollout-*.jsonl`.
- The app reads local Claude Code logs from `~/.claude/projects/**/*.jsonl`.
- Keep secrets, credentials, account details, and production access details out
  of repo documents unless the user explicitly asks for a private local-only
  note.
- Do not add network upload, telemetry, or remote sync behavior without explicit
  user approval.

## Implementation Notes

- Prefer existing single-file SwiftUI/AppKit patterns in
  `Sources/AiCliLogApp/AiCliLogApp.swift` unless a refactor is necessary.
- Use structured parsers and SQLite APIs for log/index data; avoid ad hoc string
  parsing when a structured format is available.
- Keep UI practical and dense. This is a developer utility, not a marketing
  surface.
- Verify SF Symbol names before introducing them.
- Avoid loud status colors; prefer restrained system colors and low-saturation
  state indicators.
- Search-result navigation in the conversation pane uses a `LazyVStack` inside
  `ScrollViewReader`. When jumping to a response that may be outside the current
  rendered range, do not rely on a single `scrollTo(responseID)`: first expand
  the query group and scroll to the group ID so SwiftUI instantiates its child
  views, then after a short main-actor delay scroll to the concrete message ID.
  Otherwise non-visible response anchors silently fail and the UI lands on the
  query or does not move.
